# Cognitive Bias Benchmark

This directory holds the canonical cognitive-bias stimulus set used by every
analysis workstream in the s1s2 project. The file `benchmark.jsonl` is the
primary artifact; everything else here is documentation or schema.

## Files

| File | Purpose |
|---|---|
| `benchmark.jsonl` | The benchmark itself, one JSON record per line. |
| `SCHEMA.md` | Per-field schema. The authoritative spec; the loader and validator both mirror it. |
| `README.md` | This document. |

## Schema in one paragraph

Every line in `benchmark.jsonl` is one `BenchmarkItem` with the fields
documented in [`SCHEMA.md`](SCHEMA.md). The dataclass mirroring the schema
lives at `src/s1s2/benchmark/loader.py`. The `correct_answer` and
`lure_answer` fields hold short literal strings; `answer_pattern` and
`lure_pattern` are Python regexes that match those literals (with word
boundaries where the literal is alphanumeric). Pairs are linked by
`matched_pair_id`: a single id always covers exactly ONE conflict primary
and ONE control primary, plus zero or more nested paraphrase strings on
the primary records.

## How the benchmark is built

The benchmark is generated by code, not hand-edited as a JSON file. The
canonical assembly lives in `src/s1s2/benchmark/build.py` and is invoked by:

```bash
# Either of these regenerates data/benchmark/benchmark.jsonl in place
python -m s1s2.benchmark.cli generate
python scripts/generate_benchmark.py
```

Both invocations:
1. Build the full set of items via `build_full_benchmark`.
2. Write them as JSONL.
3. Run `validate_benchmark` and exit non-zero if any check fails.

The CLI also exposes:

```bash
python -m s1s2.benchmark.cli validate     # structural validator with full report
python -m s1s2.benchmark.cli stats        # per-category counts
python -m s1s2.benchmark.cli generate     # rebuild benchmark.jsonl
```

The Hydra config (`configs/benchmark.yaml`) holds the per-category target
counts; it is read by automation but not by the CLI directly (the CLI
takes its defaults from the dataclass-level constants for simplicity).

## Composition

| Category | Pairs | Notes |
|---|---:|---|
| `crt` | 30 | 10 ratio (bat-and-ball isomorphs), 10 work-rate (widgets-machines), 10 exponential-growth (lily-pad). |
| `base_rate` | 20 | Representativeness-heuristic descriptions cueing rare professions against common ones. |
| `syllogism` | 25 | Belief-bias 2x2 (Evans 1983): 12 (valid, unbelievable) and 13 (invalid, believable) conflict cells, each with a matched (control) cell flipping validity while preserving believability. |
| `anchoring` | 15 | Numeric estimation with a misleading high anchor; controls are unanchored. Fact-based questions where the true value is well-known. |
| `framing` | 15 | Asian-disease-style life-saving scenarios with EVs constructed to match exactly. Conflict items use a gain frame; controls use an explicit-outcome frame. |
| `conjunction` | 12 | Linda-style person descriptions cueing a narrow stereotype against a broader category. Controls describe the person neutrally. |
| `arithmetic` | 25 | Multi-step integer arithmetic wrapped in a brief narrative whose surface phrasing on one step is tempting-but-wrong. Controls present the same arithmetic as a numbered step list. |
| **Total** | **142 pairs (284 items)** | Every primary has exactly one matched primary control. |

Each primary item also carries 1-4 paraphrase strings nested in the
record itself. Use `s1s2.benchmark.templates.expand_paraphrases()` at
extraction time if you need each paraphrase to be its own record.

## Provenance

| `source` | What it means |
|---|---|
| `novel` | Authored fresh for this project (the entire shipped benchmark). |
| `template` | Reserved: produced by an external template generator. |
| `hagendorff_2023` | Reserved: items adapted from Hagendorff et al. (2023) "Human-like intuitive behavior and reasoning biases emerged in large language models but disappeared in ChatGPT." |
| `adapted` | Reserved: items adapted from another published paradigm. |

The shipped benchmark is **entirely `novel`**. We deliberately do NOT
include the canonical exemplars (the bat-and-ball problem, the lily-pad
problem, the Linda problem, the Asian disease problem). Those are
available as Frederick (2005), Tversky & Kahneman (1981, 1983), and
Wason (1968) and almost certainly appear verbatim in the training data
of every model in our roster. Including them would conflate
contamination with reasoning. Instead, every item here is a structural
isomorph: the same logical form with novel surface content (cover
stories about kettles, lichen colonies, Italian relay teams, mosaic
floors, etc.).

If you find a near-duplicate of a famous exemplar in this benchmark,
file an issue.

## Contamination caveats

A novel structural isomorph is not the same as a contamination-free
stimulus. Models trained on internet text have seen many exemplars of
each paradigm and may have learned to recognise the *form* even when
the surface content is unfamiliar. We expect:

* CRT items to be partially contaminated even with novel covers because
  the bat-and-ball solution path is heavily discussed online.
* Conjunction-fallacy items to suffer the most contamination because
  Linda is one of the most-cited cognitive science examples ever.
* Belief-bias syllogisms to be the least contaminated because the
  paradigm has lower internet visibility than the others.

To measure contamination, the project ships a separate (small) set of
the canonical exemplars verbatim under the `template` source label. The
analysis pipeline compares behaviour on canonical vs novel items.

## Regenerating after a schema change

If `SCHEMA.md`, the dataclass in `loader.py`, or the validator in
`validate.py` are updated, regenerate via:

```bash
python -m s1s2.benchmark.cli generate
python -m s1s2.benchmark.cli validate     # re-confirm
pytest tests/test_benchmark.py            # round-trip checks
```

If a generator's signature changes, also update the corresponding spec
list in `src/s1s2/benchmark/build.py`.

## Loading from Python

```python
from s1s2.benchmark import (
    load_benchmark,
    iter_matched_pairs,
    filter_by_category,
    validate_benchmark,
)

items = load_benchmark("data/benchmark/benchmark.jsonl")
print(len(items), "records")

for conflict, control in iter_matched_pairs(items):
    assert conflict.matched_pair_id == control.matched_pair_id
    assert conflict.difficulty == control.difficulty

crt_only = filter_by_category(items, "crt")

report = validate_benchmark("data/benchmark/benchmark.jsonl")
assert report.ok, report.errors
```
